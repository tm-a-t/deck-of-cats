# Пираты — Deck Builder

Веб-игра на Phaser 3, пиксель-арт. `index.html` + скрипты в `js/`.

## Правила

- Жанр: deck-building, но вместо карт — пираты (члены команды).
- **Раунд**: корабль с 5 пиратами (из колоды) прибывает к случайному острову.
  1. **Высадка**: игрок отправляет 0–2 пиратов на остров (по одному, клик = отправка). Действие мгновенное.
  2. **Корабль**: оставшиеся пираты выполняют корабельное действие одновременно.
  3. **Магазин**: игрок покупает новых пиратов за коварство (☠️). Коварство сбрасывается, ресурсы остаются.

## Стартовая колода (10 пиратов)

| Тип | Кол-во | Остров | Корабль |
|-----|--------|--------|---------|
| Дровосек | 3 | 90% → 1🪵 | 3🪵 → 2☠️ |
| Шахтёр | 3 | 90% → 1🪨 | 3🪨 → 2☠️ |
| Искатель | 2 | 50% → 1🪙 | 3🪙 → 5☠️ |
| Лентяй | 2 | не может | → 1☠️ |

## Острова (случайный каждый раунд)

- Лесной — x2 к добыче 🪵
- Скалистый — x2 к добыче 🪨
- Остров сокровищ — x2 к добыче 🪙

## Промах на острове

Если пират не добыл целевой ресурс → получает 1 случайного другого ресурса (или карту сокровищ 🗺️ с шансом 1%). Карта автоматически тратится при отправке искателя — даёт +30% к шансу золота.

## Спрайтшит

`assets/pirates.png` — 48×40 px, сетка 5×4, фреймы 8×8 px, spacing 2px, margin 0. Всего 20 фреймов.

## Архитектура

- Разрешение: 960×1440 (Scale.FIT)
- Глобальное состояние в объекте `G`
- Одна Phaser-сцена `GameScene`
- UI перерисовывается целиком через `renderAll()` при каждом изменении стейта
- Контейнеры: `top`, `island`, `phase`, `hand`, `btn`, `shop`, `tip`, `fx`

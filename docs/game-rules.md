# Deck of Cats — Правила игры

Этот файл в `docs/` описывает текущий геймплей по коду в `js/`. Корневой `rules.md` остаётся главным source of truth, поэтому gameplay-изменения нужно отражать в обоих файлах.

## Ключевая идея

- Это deck-builder про pirate cats.
- Базовые ресурсы: `🪵 wood`, `🪨 stone`, `🪙 gold`, `☠️ enthusiasm`.
- Островные действия сейчас детерминированы: в коде больше нет старых `90%`-шансов, промахов, карт сокровищ, временных мечей или постоянных пушек.
- Корабельные эффекты теперь в основном производят `☠️` и личное оружие пиратов. Оружие остаётся на пирате, пока он не покинет команду, и именно оно определяет поведение на абордаже.

## Старт обычного забега

- Стартовая колода: 10 пиратов.
- Стартовые ресурсы и `☠️`: `0`.
- Стартовая рука: до 5 пиратов.
- Стартовый магазин: 4 случайных пирата из `SHOP_POOL` со стоимостью не выше `3`.
- Если deck пуст, весь discard тасуется обратно в deck. Если карт меньше 5, новая рука просто получается меньше.

| Pirate | Кол-во | Остров | Корабль |
|---|---:|---|---|
| Rigger | 4 | 🪵 | 4🪵 → ☠️☠️ |
| Ballaster | 4 | 🪨 | 4🪨 → ☠️☠️ |
| Armsman | 2 | 🔨 | — |

## Цикл обычного забега

### 1. Карта

- Забег начинается в фазе `map`.
- Если доступен только 1 следующий узел, он выбирается автоматически.
- Если доступно 2+ узла, игрок выбирает маршрут на карте.
- При выборе узла `round` увеличивается на 1, `☠️` сбрасывается в `0`, ресурсы сохраняются.
- Вне фазы `map` карта работает как просмотр; выбирать узлы можно только в `phase = map`.

### 2. Островный раунд

- Если выбран island node, раунд входит в фазу `sending`.
- По умолчанию можно отправить до 2 пиратов.
- `Port Island` даёт `+1` к лимиту, то есть максимум 3 отправки.
- Отправка анимируется, но следующего пирата можно отправить сразу, не дожидаясь конца предыдущего эффекта.
- Каждый отправленный пират моментально резолвит свой island-эффект после приземления.
- Игрок может остановиться раньше лимита кнопкой `End`. Если лимит уже заполнен, кнопка меняется на `Work on Ship`.
- Пирата с island-конверсией нельзя отправить, если не хватает входного ресурса.
- `Bosun` вообще не может идти на остров.
- На `Siren Island` пират сначала выполняет island-эффект, а потом навсегда удаляется из команды.
- Бонус острова удваивает только выходы по соответствующему ресурсу (`wood`, `stone`, `gold`). Он не удваивает guaranteed-эффекты, `☠️`, recall/exile и выдачу оружия.

### 3. Корабельный раунд

- После `End` или `Work on Ship` все неотправленные пираты резолвят ship-эффекты по порядку текущей руки.
- Ship-эффект может:
  - ничего не делать;
  - тратить ресурсы и производить ресурсы, `☠️` и оружие;
  - удалить самого пирата (`get lost`);
  - запустить выбор пирата на изгнание из общей команды.
- Если на ship-эффект не хватает стоимости, он просто не срабатывает.
- `Cutthroat` при успешной оплате переводит игру в фазу `removing`: игрок выбирает 1 пирата из `allCrew`, которого нет в текущей руке, и навсегда удаляет его.
- Если ship-эффект выдал оружие, оно назначается сразу же, по одному, на любого пирата из текущей руки, включая уже отправленных на остров. Новое оружие заменяет старое.

### 4. Магазин

- Магазин открывается только после island-раундов. После абордажа магазина нет.
- В магазине всегда 4 слота. Стартовые пираты (`Rigger`, `Ballaster`, `Armsman`) туда не входят.
- `randomShopType(round)` берёт случайного пирата из `SHOP_POOL` с ограничением `cost <= max(3, round + 1)`.
- Стартовая витрина создаётся как `initialShop(4, 0)`.
- Мгновенное пополнение после покупки использует текущий `G.round`.
- Переход к следующему раунду (`Continue`) удаляет левый слот, сдвигает остальные влево и докидывает нового пирата уже по правилу для следующего раунда: `randomShopType(G.round + 1)`.
- Купленный пират сразу попадает в `discard`, а не в руку.
- За один визит можно купить сколько угодно пиратов, пока хватает `☠️`.
- При `Continue`:
  - рука уходит в discard только теми пиратами, которые всё ещё есть в `allCrew`;
  - изгнанные и `get lost` пираты не возвращаются;
  - `☠️` сбрасывается в `0`;
  - тянется новая рука до 5 карт;
  - игра возвращается в `map`.

### 5. Абордажный раунд

- Ship nodes переводят игру сразу в `phase = boarding`.
- Старой проверки "суммарная сила команды >= сила корабля" больше нет.
- У ship node всё ещё хранится числовое поле `strength`, но текущий бой использует сгенерированную вражескую boarding party, а не прямое сравнение силы.
- Перед боем текущая рука складывается в компактную формацию из 3 рядов.
- Игрок может перетаскивать пиратов между рядами и менять их порядок слева направо.
- Стартовая расстановка игрока:
  - вооружённые ranged-пираты идут в средний ряд;
  - все остальные стартуют в переднем ряду.
- Кнопка `Fight!` запускает автосражение.
- Базовые боевые статы всех пиратов до оружия одинаковы: `9 HP`, `3 damage`, `1350 ms attack`, melee-атака из переднего ряда.
- Боевые потери не постоянны: павшие пираты не удаляются из колоды навсегда.
- После победы вся текущая рука уходит в discard, тянется новая рука, и игрок возвращается на карту.
- После поражения показывается экран `Game Over`.
- Победа над последним кораблём на слое 29 сразу завершает забег экраном `Victory`.

## Weapons

- Выданное оружие не попадает в общий инвентарь.
- Каждая единица оружия сразу назначается конкретному пирату из текущей руки.
- Если у выбранного пирата уже было оружие, старое заменяется новым.
- Оружие остаётся на пирате, пока этот пират не покинет команду.

| 🔨 Hammer | Melee. `+4 HP` |

## Boarding Combat

- На поле до 3 рядов с каждой стороны: front, middle, back.
- Front row всегда означает первый живой ряд. Если ряд погибает целиком, следующие ряды автоматически считаются передними.
- Melee-бойцы могут атаковать только из текущего front row.
- Ranged-бойцы могут атаковать из любого живого ряда.
- Каждый боец получает случайную стартовую задержку `80–260 ms`.
- Во всём бою только одна новая атака может стартовать каждые `300 ms`.
- Боец не может начать собственную атаку, пока в него сейчас прилетает чужая атака.
- Базовое правило цели `frontBand`:
  - если атакующий стоит на позиции `X` из `N` живых союзников в front row, а у противника `M` живых целей в его front row, то он выбирает случайную цель из диапазона `floor((X-1)*M/N)` ... `ceil(X*M/N)-1`.
- Multi-target атаки по ряду уменьшаются на `2 damage` по каждой цели, если в этом ряду жив хотя бы один `Shellback`. Минимум всё равно `1 damage`.
- `Powder Bomber` взрывается сразу после смерти и наносит `4 damage` всему front row игрока.
- Bleed тикает во время автосражения и может убивать между обычными атаками.

### Скейлинг вражеских боёв

- `Boarding 1`: ровно 3 врага, из них 1 strong и 2 weak.
- `Boarding 2`: ровно 3 врага, из них 2 strong и 1 weak.
- `Boarding 3`: ровно 4 врага, обычно 2 strong и 2 weak. На этом этапе разблокируется `Netter`.
- `Boarding 4`: ровно 4 врага, обычно 3 strong и 1 weak.
- `Boarding 5` и `Boarding 6`: ровно 5 врагов, все strong. На `Boarding 5` разблокируется `Flint Duelist`.
- У каждого ship node заранее генерируется blueprint с главным archetype и коротким `encounterDesc`, который игра показывает как подсказку перед следующим боем.
- В расстановке врагов код старается держать melee впереди, а ranged глубже; "пустой front row с живыми врагами позади" не допускается.

## Карта и победа

- Всего в run `30` слоёв и `6` ship nodes.
- Ранний блок:
  - `layers 0–3`: один линейный путь из island nodes;
  - `layer 4`: первый ship node;
  - `layers 5–8`: три параллельных непересекающихся island path;
  - `layer 9`: второй ship node.
- На ранних island layers используются только `Forest Island`, `Rocky Island` и `Port Island`. `Treasure`, `Skull` и `Siren` там не появляются.
- Начиная с `layer 10`, обычный island layer содержит `2–3` узла.
- Начиная с `layer 10`, `Siren Island` попадает в пул слоя с шансом `50%`; остальные islands могут появляться всегда.
- После `layer 9` ship nodes стоят на `layers 14, 19, 24, 29`.
- На поздней карте каждый обычный узел соединяется с `1–2` узлами следующего слоя; код гарантирует достижимость каждого узла следующего слоя и старается не делать пересечений линий.
- Ship nodes по-прежнему получают числовые `strength`: `6`, `8`, `11`, `14`, `17`, `21`, но текущий combat напрямую эти числа не использует.
- Победа в обычном забеге наступает после выигрыша `Boarding 6` на `layer 29`.

## Battle Test

- `Battle Test` из меню не использует карту, магазин, deck и discard.
- Тест стартует с 5 случайными пиратами из всего `TYPES`.
- `round` и номер абордажа случайно выбираются от `1` до `6`.
- Команда получает от `1` до `5` случайных оружий через `rollWeaponKeys(..., { ensureDistinct: true })`.
- После боя:
  - `Repeat` запускает точно тот же состав, тот же enemy party и ту же pre-fight расстановку;
  - `Another Battle` полностью бросает всё заново.
